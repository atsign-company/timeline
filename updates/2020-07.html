@ Attestations

We have been working with SunStone Defense to secure our registrar site from attacks, including botnets. A happy side effect of this is that we will have our first attestation (isHuman) that @sign owners can use as proof that the @sign was created by a real human.

We have made some additional progress on attestations, it turns out there are national non-profits, such as the American Association of Motor Vehicle Administrators and the National Association for Public Health Statistics and Information Systems (NAPHSIS) that aggregate driver licenses, birth certificates, social security numbers and more. We are in early discussions with NAPHSIS and they are working on a proposal for us to attest for age using their Electronic Verification of Vital Events database.

@ Core Protocol

Good news! The core team has finished producing both a server build and a client SDK (Software Developer Kit) with the complete set of verbs implemented and working in unison. We are now working on migrating our existing infrastructure and applications to the new codebases.

The net effect of all this work is that developers of applications using the @ protocol and the SDK only need to know two things. First, how to include the SDK in their code, which only requires a couple of lines of code, and second, what verbs they want to use. The current verbs are very easy to understand: “update”, “lookup”, “delete”, “scan”, “stats”,”sync” are all pretty self-explanatory for the developer. 

We also have a few really interesting updates to verbs. For example, the scan verb can now accept a regular expression and the sync verb now syncs without the need for timestamps.

The interns have been instrumental in creating a new API that provides an interface between the registrar site and our infrastructure services. This will be a key part of the activation process together with the @me application that binds the @sign on a mobile device to the cloud microservice with full control literally in the hands of the owner of the @sign.

It is so satisfying to see all these components coming together and working together as a system — that's how protocols should work, but it is nonetheless very exciting.
