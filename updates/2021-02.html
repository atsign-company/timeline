Now for the Techies!

For developers: Running an open-source project means that we should be using the open-source code in production and this month we can say we are. The production environment now is built on the public source codebase. So if you see an issue in production then any developer can help fix it!

Having demonstration code is always useful (and we have some example code in the demo repo), but what is even nicer is a full application. So, we have open-sourced the @mosphere codebase which can be found here. More applications will go open-source in the future, and the next one is likely to be @persona.

Using multiple @signs in a single @pp was always possible, but now it has been implemented in the @persona app, allowing you to manage all your @signs on atsign.directory.

@operations: We have had a good month operationally, and we did hit the magic 99.999% overall uptime for our core infrastructure (status.atsign.com), although because it's a moving target we do of course get hit with the odd problem.

Our website was hit with an outage for 3 minutes where responses were slower than expected. We use kinsta.com for the website and they continue to provide excellent service, but together we could not find the cause of the slowness for those three minutes.

We also did have an outage on the registrar website where people could not register new @signs. That issue was a result of human error on our part. We fixed it quickly, and anyone who had a problem was contacted personally.

@core protocol: 

We also have a number of things in the test environment that will be published soon. These include a completely re-engineered notification subsystem, the migration of the “keys” file from a .zip file to a .atKey file, and a much easier onboarding process. 

What we have already published is a newly written PKAM utility. This allows you to login by hand into your @server by providing you with the cryptographic answers to login with the pkam: verb. You can find that code in the at_tools repo.
