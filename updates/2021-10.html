Now, on to the techy stuff:

  • 100% uptime for the latest hackathons - After the outage, the later hackathons and events did not encounter any fundamental blockers for the developers that participated in them. This is a great demonstration that the hard work to improve the reliability and usability of the SDKs, widgets, and tools is paying off.
  • Google’s Dart team recognized Chris Swan’s work - Chris created a multi architecture Docker build for the Dart language. Chris was asked to raise a pull request to the Dart language GitHub repository so his changes could become part of the official Google build. https://github.com/dart-lang/dart-docker/pull/53 
  • Outage 5:25 PM Thurs Sept 21st till 2:20PM Friday Oct 2nd PDT -

During this time many secondaries were down, and at one point all secondaries were down as we reset the cluster’s networking configuration.

  • What happened - A combination of two things. First, a new Secondary server build, which moved from a purely in-memory database to an in-memory index and data on disk strategy. Second, an increase in the number of secondaries, due to the increased traction. The combination of the two resulted in secondaries failing and restarting, which in turn put pressure on the infrastructure. We saw load averages that normally settle in at 1 to 3 bouncing in the high teens during the first few days of running the new code, then on Thursday sky rocketing to over 100 and the Docker swarm cluster was not able to recover without intervention.
  • How we reacted - Clearly we had to revert our code back, so the core team produced a build that went back to the in-memory model but kept all the bug fixes from the previous build. We also took the opportunity to upgrade and increase the number of worker nodes from 6 to 12. We knew that the latest version of Docker had an MTU (Maximum Transfer Unit) issue on GCP (GCP interfaces are set to 1460 not 1500!). To fix this, we changed the MTU of the Docker swarm Ingress network and took the cluster down then put it back up.
  • The platform has been stable since the outage - We have re-looked at our decision to move completely to index in-memory and data on disk. The next build will have some databases fully in-memory and others just the index.

 

Wow, can’t believe that is just one month, this team is unstoppable and I feel so fortunate to be able to work with them!
